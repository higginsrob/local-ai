## Key Features

### ğŸ”’ Safety Checks
- **Main Branch Verification**: Tag must be on the HEAD of main branch
- **Annotated Tag Verification**: Only annotated tags (not lightweight) trigger releases
- **Version Pattern**: Tag must start with `v` (e.g., v0.0.4)

### ğŸ“ Automatic Documentation
- GitHub release notes are automatically populated from the tag message
- Encourages detailed release descriptions

### ğŸ”„ Two-Stage Process
1. **verify-and-release**: Validates and creates the GitHub release
2. **publish**: Builds, tests, and publishes to NPM (only if verification succeeds)

### âš¡ Fast Feedback
- Validation happens first (before long build/test process)
- Immediate failure if tag is invalid
- Clear error messages in GitHub Actions logs
